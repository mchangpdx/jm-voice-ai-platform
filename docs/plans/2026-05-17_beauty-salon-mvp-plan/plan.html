Phase 1 — Vertical Template Framework 표준화 (3-4일) ⭐ 핵심

목표 (확장성 핵심)

이번 sprint의 가장 큰 가치. 한 번만 잘 만들면 모든 future vertical 추가가 yaml만 작성하면 가능. 9-layer Vertical Template Framework 표준 설계 + 구현.

Task

#작업시간산출물
1.1 9-layer spec 문서 작성 (각 layer schema + 필수 필드) 1일 backend/app/templates/_base/spec.md
1.2 vertical_kind 분류 정의 (order / service / service_with_dispatch) 0.5일 _base/vertical_kinds.yaml
1.3 yaml validator (loading 시 자동 검증, 명확한 에러) 0.5일 _base/validator.py + unit tests
1.4 기존 cafe template 9-layer 마이그레이션 (회귀 안전) 0.5일 templates/cafe/ 재구성
1.5 pizza / mexican / kbbq 마이그레이션 (동일 패턴) 1일 4 vertical 모두 9-layer 통일
1.6 build_system_prompt vertical-aware (intake_flow.yaml 읽음) 0.5일 voice_websocket.py 갱신

9-Layer 표준 (재명시)

Layer파일명역할
1safety_rules.yaml위험 매트릭스 (음식점 = allergen, beauty = chemical sensitivity, home = hazard)
2service_categories.yaml서비스 카테고리 (메뉴 또는 서비스 항목)
3option_groups.yaml옵션 변형 (modifier 또는 service add-on)
4system_prompt_base.txtvertical 페르소나 + 룰
5intake_flow.yaml ⭐ NEWPHASE FLOW 정의 (음식점 5-phase / service 5-phase 다름)
6scheduler.yaml ⭐ NEW슬롯 구조 (테이블 / stylist / technician)
7emergency_rules.yaml ⭐ NEW긴급 처리 (severe allergy / 24h cancel / 가스누출)
8crm_followup.yaml ⭐ NEWvertical 별 followup (the usual / 6-week refresh / 정기 점검)
9pricing_policy.yaml ⭐ NEW가격 정책 (메뉴 / flat-rate / hourly+fee / membership)

Exit Criteria

  • ✓ 9-layer spec 문서 confirm
  • ✓ 4 기존 vertical 모두 9-layer로 마이그레이션
  • ✓ build_system_prompt yaml-driven
  • ✓ 200+ unit tests + 라이브 통화 (cafe, pizza, mexican 1건씩) 통과
  • ✓ 회귀 없음 — 기존 매장 동작 변경 없음

Phase 4 — Beauty Template 자료 작성 (2일)

Task

backend/app/templates/beauty/
├── safety_rules.yaml          — perm/coloring 화학물질, 임산부 주의, 두피 sensitivity
├── service_categories.yaml    — haircut / coloring / styling / nails / facial / brows
├── option_groups.yaml          — 길이(short/medium/long), 추가(blowdry, deep conditioning, treatment)
├── system_prompt_base.txt     — Luna persona 강화 (현재 1줄 → 실용 paragraph + 룰)
├── intake_flow.yaml           — INTAKE → SERVICE_SELECT → STYLIST → TIME_SLOT → CONFIRM
├── scheduler.yaml             — 30/45/60/90/120분 슬롯 분류, stylist별 available
├── emergency_rules.yaml       — 24h cancellation 정책, 응급 변경
├── crm_followup.yaml          — 6-week haircut refresh 알림 (returning customer)
└── pricing_policy.yaml        — flat-rate per service + add-on 합산, 팁 룰

  • Vietnamese: nail salon 특화 (manicure/pedicure)
  • Japanese: シャンプー, カット, カラー
  • Chinese: 剪发, 染发, 烫发

3KO"내일 2시에 헤어컷 예약 가능해요?"한국어 흐름 정상

6EN"Cancel my 2pm appointment"cancel_appointment (24h 이상)
7EN24h 이내 취소 시도manager 안내 / 정책 룰 작동
8EN영업시간 외 통화voicemail or manager handoff
9KO"이번 주 가능한 시간 알려주세요"list_stylists + slot 안내
10ENReturning customer (CRM Wave 1)"Welcome back, Michael" + 6-week refresh 제안
