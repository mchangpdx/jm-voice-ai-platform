3. JM의 Defensible Edge (방어 가능한 우위)

현재 보유

  1. Modify/Cancel 풀 사이클 (B1+B2 라이브) — Maple docs에 부재
  2. 3중 hallucination dedup (5s/8s/8s) — Maple은 단순 confirmation gate만
  3. PostgreSQL RLS Mandatory — Maple 무증거
  4. Caller-ID server-side override — Maple 약함
  5. Loyverse 라이브 검증 — Maple 미지원 (한식당 시장 wedge)
  6. Per-store granular overrides — loman.ai가 Maple 약점으로 지적
  7. Audit trail (bridge_transactions) — Maple 미문서화
  8. 추상 POSAdapter 아키텍처 — 새 POS 추가 cost ↓

잠재 우위 (개발 필요)

  1. 한국어 native — 시장에 한국어 식당 voice AI 존재 안 함
  2. Quantic 화이트라벨 — 통합 시 동급
  3. KakaoTalk SMS bilingual — 한국 식당 특화

4. JM의 즉시 격차 (Pre-Launch Blocker)

  1. B3 modify_reservation + B4 cancel_reservation (Maple 8/10, 우리 0/10)
  2. B5 Allergen Q&A (Maple 5/10, 우리 0/10) — 한식 알러젠 더 복잡
  3. OpenTable 통합 (한식당 OpenTable 사용률 낮으니 영향 적을 수도)
  4. Manager Dashboard / Analytics UI (Maple 7/10, 우리 1/10) — 영업 결정타
  5. Call transfer to staff — 복잡 요청 핸드오프

5. 전략 제언

Wedge 1: 한국어 식당 vertical

  • Maple 한국어 0/10, OR/CA/NY/NJ Korean restaurant 수천 개
  • KAGRO + Korean Chamber of Commerce 채널
  • KakaoTalk 결제 알림, 한국어/영어 bilingual 자동 전환
  • 한식 메뉴 modifier (반찬 옵션, 매운 정도, 고기 부위)

Wedge 2: Loyverse 시장

  • Loyverse 글로벌 50만+ 매장. Maple 미지원
  • 우리가 유일 옵션 — 한국 + 동남아 + 중남미 매장 타겟

Wedge 3: Modify/Cancel 안정성

  • Maple 문서에 modify_order/cancel_order 부재
  • 영업 demo에서 라이브 modify 시연으로 차별화

Wedge 4: RLS + Audit (security-conscious)

  • 호텔 체인, 프랜차이즈 본사 영업 시 결정타
  • Maple은 multi-store override 약점 인용됨

즉시 닫아야 할 갭

  1. B3/B4/B5 완성 (4–6주)
  2. Manager Dashboard MVP (2–3주) — call log + order metrics + missed calls
  3. Twilio TCR 승인 진행 (외부 의존)
  4. 한국어 system prompt + Korean greeting (1주) — 큰 변경 없음

가격 전략

  • Maple: $240–$299/월 base + 모듈 add-on
  • JM: 한국 식당 시장에 맞춰 $199/월부터 또는 per-store flat + Loyverse migration credit
  • Free trial 14일 (Maple 미제공) — Capterra 0 reviews를 우리는 reviews로 채우기
