C-ALLERGEN. Allergen Q&A 1:1 심층 비교 (2026-05-02 v3 신설)

근거: B5 라이브 검증 2건 (call_6d69342b Scenario A 6 hints / call_4630b947 Scenario B Tier 3) + Maple 공식 마케팅 (maple.inc/product, Capterra) + 3rd-party (Hostie 2025, Loman 2026, Kea 2026) + Hostie 3-tier framework reference. Maple architecture는 비공개 — 공개 클레임 + 업계 reference로 추정.

0) 핵심 발견 (executive summary)

JM이 Maple 우위 7개 / 동급 2개 / Maple 우위 5개. JM의 우위는 안전성·검증가능성·운영자 통제; Maple의 우위는 기능 폭 (cross-contam, substitution, nutritional DB, live transfer). v1 ship 시점에서 JM은 "가장 안전한 allergen Q&A" 포지셔닝, Maple은 "가장 풍부한 allergen UX" 포지셔닝.

1) Maple의 공식 클레임 (마케팅 그대로 인용)

출처클레임구체성

2) Maple의 보고된 약점 (3rd-party)

  • "Confidence overruns" (Loman 2025-12) — allergen 답변에 disclaimer 부족, 자체 LLM 지식으로 over-confident하게 답하는 사례 보고. JM은 spec Decision #3 + I0 invariant로 코드/스크립트 수준에서 차단.
  • POS-side allergen 메타 부재 (Loman) — POS (Toast/Square/Quantic)는 allergen 컬럼 없음. Maple도 manager dashboard 수동 입력 의존. JM도 동일 (수동 SQL backfill v1) — 동급.
  • Multi-store override 약점 (Loman) — franchise 본사 vs 매장별 메뉴 차이 처리 약함. JM의 store_id RLS + per-store backfill로 우위.
  • Disclaimer 워딩 비공개 — 검증 불가. JM은 spec §7 / order.py에 7개 hint script 모두 코드로 고정 ("per our kitchen records" 등).
