Part 2 — Twilio 번호 풀 자동 할당 메커니즘

핵심 답: 두 방식 모두 가능. Twilio가 제공하는 3가지 API를 조합하면 완전 자동화.

Twilio 자동화 가능 API 3가지

A) Search Available Numbers — "어떤 번호가 살 수 있나요?"

GET /2010-04-01/Accounts/{Sid}/AvailablePhoneNumbers/US/Local.json
    ?AreaCode=503&Contains=AI&SmsEnabled=true&VoiceEnabled=true

응답: [
  { "phone_number": "+15031234567", "friendly_name": "(503) 123-4567", ... },
  { "phone_number": "+15032345678", ... },
  ...
]
  • 지역번호 (area code), 패턴 ("AI" 포함 등), 능력 (voice/sms) 필터링 가능
  • 미국, 캐나다, 영국, 호주 등 지역별 구매 가능

B) Buy a Number — "이 번호 살게요"

POST /2010-04-01/Accounts/{Sid}/IncomingPhoneNumbers.json
data: { PhoneNumber: "+15031234567" }

응답: { "sid": "PN123abc", "phone_number": "+15031234567", ... }
  • 비용: $1/월 + $0.0085/분 통화 (시내 번호)
  • Toll-free (1-800): $2/월 + $0.013/분
  • 즉시 활성화 (몇 초 내)

C) Update Voice Webhook — 우리 A5 (이미 구현)

POST /2010-04-01/Accounts/{Sid}/IncomingPhoneNumbers/{PhoneSid}.json
data: { VoiceUrl: "https://api.jmtechone.com/twilio/voice/inbound", VoiceMethod: "POST" }

두 가지 운영 방식 비교

방식장점단점
1) 정적 할당 (Static)
매장이 자기 번호 직접 가져옴		 매장이 원하는 area code, vanity number 가능 영업/온보딩 시간 5–10분 추가, 매장 판단 필요
2) 사전 구매 풀 (Pool) 추천
우리가 미리 사놓고 자동 배정		 영업 작업 0초 (위저드 클릭만), 매장 등록 즉시 통화 가능, 일관된 number format 미사용 번호도 월 $1씩 비용 (30개 = $30/월)
3) 하이브리드 최선
풀 자동 + 옵션 search&buy		 기본은 풀 자동 / 특정 area code 원할 시 즉시 search&buy 위저드 UI 옵션 3가지 추가 필요

Part 3 — 권장 운영 방식 (하이브리드)

DB 스키마

CREATE TABLE twilio_number_pool (
  phone_number   TEXT PRIMARY KEY,           -- +15035551234
  sid            TEXT NOT NULL,              -- PN123abc...
  area_code      TEXT,                       -- 503
  status         TEXT NOT NULL,              -- 'free' | 'assigned' | 'reserved'
  assigned_store TEXT REFERENCES stores(id),
  purchased_at   TIMESTAMPTZ,
  assigned_at    TIMESTAMPTZ
);
